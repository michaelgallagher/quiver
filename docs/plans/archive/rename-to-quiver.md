# Rename: "Prototype Flow Map" → "Quiver"

Status: complete (2026-05-18) — 40 files changed; CLI runs as `quiver`, full
module graph loads, all injected identifiers renamed, archive left untouched.

Decision: **full rename**, with the config loader **supporting both** the new
`quiver.config.*` filenames and the legacy `flow-map.config.*` / `.flow-map.json`
names as a fallback, so existing prototype repos keep working.

## Token map (applied via scripted replace, longest-first)

| From | To | Scope |
|---|---|---|
| `injectFlowMapRouteHandler` | `injectQuiverRouteHandler` | exported JS fn (swift-injector, swift-spike-runner) |
| `flowMapSubNavDestination` | `quiverSubNavDestination` | injected Swift helper |
| `flowMapSubDestination` | `quiverSubDestination` | injected Swift helper |

| `flow-map.config` / `.flow-map.json` | `quiver.config` / `.quiver.json` | everywhere **except** the config loader |

## Manual / non-mechanical changes

- `src/flow-map-config.js` → `git mv` to `src/quiver-config.js`; update the two
  `require("./flow-map-config")` / `require("../src/flow-map-config")` call sites.
- Config loader filename arrays: primary `quiver.config.yml|yaml|json`, `.quiver.json`;
  keep `flow-map.config.yml|yaml|json`, `.flow-map.json` as fallback (no break).

## Excluded (intentionally untouched)

- `docs/plans/archive/**` — historical records, left as written.
- `.claude/settings.local.json` — historical permission grants, not tool identity.
- `node_modules/`, `package-lock.json` (regenerated on install).
- Generated `flow-map-output/` on disk — regenerated as `quiver-output/` next run.
- Stale `~/.cache/prototype-flow-map/` — orphaned, harmless; new runs use `quiver/`.
- The repo folder name itself and the `.flow` scenario file extension.

## Follow-ups for the user (out of band)

- Rename the repo directory `prototype-flow-map/` if desired (affects shell/IDE only).
- `npm install` to regenerate `package-lock.json` with the new package name.
- Delete the old `flow-map-output/` and `~/.cache/prototype-flow-map/` if you want.
